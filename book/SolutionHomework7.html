8.6 Task 3 (1 pt)

In 1-5 sentences, describe in your own words the difference in output between the left and inner joined data between df1 and df2 below. Essentially, answer the following: what is left_join() doing compared to inner_join()?

Your response: left_join() appends data from the 2nd data frame (if there is a match) to the 1st data frame and if there are no matches, new column fields are left blank (NAs). inner_join() only keeps rows where there is a match between the two data frames, thereby excluding non-matching rows.

8.11 Task 7 (1 pt)

Explain in detail why the table outputs for Task 4 and Task 6 are different or the same. In this case, does the order matter at all for left_join()?

Your response: The order matters. The difference arises because the order of the data frames determines which data frame’s rows are fully preserved in the output (i.e., rows from the first data frame fed into left_join), then columns/variables from the 2nd data frame is then added to the first data frame. Unmatched data are then assigned NAs with respect to the additional variables or columns added from the 2nd data frame. Another framing: in Task 4, we assign an employee to each department, while in Task 6, we assign a department to each employee.

8.18 Task 13 (1 pt)

In detail, what does the join you performed in Task 12 achieve? Explain the join and output in detail. What type of join did you perform (i.e., filtering, mutating)?

Your response: Performed a filtering join via semi_join() as semi_join() doesn’t add additional variables. In semi_join(), rows from the first data frame are only kept if and only if their join key(s) exists or have a match in the 2nd data frame.
