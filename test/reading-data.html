5.5 read.csv(): Reading tabular files

Tabular files (e.g, USASpending .csv data) are typically read as data frame objects within the R environment.

In R, you are able to read csv files into data frame by running this script:

awards_data <- read.csv("~/USASpending/FY2019_073_Assistance_Full_20230309_1.csv")

How to interpret Line 11 code: The script above uses the built-in read.csv() R function to locate and read the FY2019_073_Assistance_Full_20230309_1.csv file. The <- operator then assigns the tabular data frame data to a variable named awards_data.

Once awards_data is created, you will see the awards_data under the environment pane:

After the creation of awards_data, we can use other R functions to inspect basic properties of this data frame object.

For this task, we may use either unique() or table().

5.10 File Path in Windows (Advanced)

A file path is the complete location of where a file is located.

To load .csv or Excel (.xls, .xlsx) files into R, you must know the file path to help R locate the relevant file(s) and the information contained within them.

In Windows, the easiest way to get the full file path of a tabular data file is by opening the file in Excel. After the relevant file is open in Excel, click File which should open a side bar on the left. You then need to click Info which should enable the appearance of a Copy local path icon which you need to click. If you were to right click and paste the copied information, you may see an information similar to the one below:

C:\Users\user\OneDrive - U.S. Small Business Administration\Documents\USASpending FY21 Data.xlsx

However, R reads files using forward slash (/) which is the norm in non-windows environment (e.g., Unix, Linux, Mac OSX). So when working with file paths in R, you need to replace the backward slash (\) with forward slash.

When working with R, this file path is the appropriate file path after replacing \ with /:

C:/Users/user/OneDrive - U.S. Small Business Administration/Documents/USASpending FY21 Data.xlsx

When loading data into R via functions such as read.csv(), you may feed this entire path (surrounded by quotation marks as you did in line 11) as an alternative.
