After opening the R file, run line 2 within the script which is:

SBAGrantsTotal <- read.csv("~/USASpending/SBAGrantsYearlyTotal.csv")

As a reminder from the previous sections, this script reads the SBAGrantsYearlyTotal.csv file you saved under the USASpending folder you created. This data frame is then saved as SBAGrantsTotal, which should appear in your Environment pane.

7.2 barplot() for bar plots or bar charts

The barplot() function, as implied by the name, creates bar plots in R.

Before moving forward, let us load the SBAGrantsYearlyTotal.csv file as SBAGrantsYearlyTotal by running line xxx. Also run line xxx to preview the data.

head(SBAGrantsTotal)
##   fiscal_year total_obligations obligations_in_millions
## 1        2016         161990239                  161.99
## 2        2017         202550783                  202.55
## 3        2018         210945448                  210.95
## 4        2019         201145377                  201.15
## 5        2020         453675554                  453.68

SBAGrantsTotal contains three variables: fiscal year, total_obligations, and obligations_in_millions. obligations_in_millions is simply another version of total_obligations but in millions (more on this shortly).

To create a barplot, let us run line 10. For the height argument, we are telling R to use the total_obligations variable from SBAGrantsTotal by specifying height = SBAGrantsTotal$total_obligations. Meanwhile, setting names.arg = SBAGrantsTotal$fiscal_year instructs R to label the x-axis or groupings using the fiscal year variable.

barplot(height = SBAGrantsTotal$total_obligations, names.arg = SBAGrantsTotal$fiscal_year)

After running the script, the plot should appear in your plot pane that should look similar to this. Please note that there is an ‘Export’ button at the top of this pane that will allow you to save your visualizations as PDF or images (e.g., .jpg. .png). Try using the Export button for practice.

Preview of the Environment Pane after creating loans_data_fy19

Important Note: Remember that R, as its core, is a statistical computing language. So as you may have noticed, the obligation dollars in the script we just ran were abbreviated in scientific notation due to the large numbers involved. There are few ways to work around this. However, one simple work around is to feed R with smaller numbers. For example, the obligations_in_millions variable are much smaller in numerical value as the sums are in millions instead (i.e., 161990239 161.99).

Now, let us redo the barplot but using the obligations_in_millions variable instead. Let us run line 14 this time, but with two additional arguments (i.e., xlab, ylab) to label the respective axes and one other argument (i.e., col) to color the bar plots.

barplot(height = SBAGrantsTotal$obligations_in_millions, names.arg = SBAGrantsTotal$fiscal_year,
        xlab = "Fiscal Year", ylab = "Obligated SBA Grants $ (in millions)", 
        col = "darkorange1")

## Practice

Within the script we just ran from line 14, we specified an col = "darkorange1" argument.

For reference on what other colors you could specify, please visit https://r-charts.com/colors/ and see what other colors are available. For practice, try rerunning the line 14 script after specifying a different color that interest you!

7.3 plot() for scatter and line plots

If you are interested in representing the grants data as scatter plot, you accomplish it by running line xxx from the script file. In fact, the arguments and structure are fairly similar to barplot().

plot(y = SBAGrantsTotal$obligations_in_millions, x = SBAGrantsTotal$fiscal_year,
     xlab = "Fiscal Year", ylab = "Obligated SBA Grants $ (in millions)")

Now, let us create a line plot using the same underlying data–to do so is simple! All we need to do is use the same code as the scatter plot examine with the addition of the type = "l" argument within the script. Now, let us run line 23 of the script and see the visualization created:

plot(y = SBAGrantsTotal$obligations_in_millions, x = SBAGrantsTotal$fiscal_year,
     xlab = "Fiscal Year", ylab = "Obligated SBA Grants $ (in millions)",
     type = "l")
